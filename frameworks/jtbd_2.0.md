# JTBD 2.0 Methodology

## Evolution from JTBD 1.0

JTBD 2.0 emphasizes evidence-driven discovery over theoretical frameworks.

## Key Differences

- JTBD 1.0: Top-down, framework-prescribed job hierarchies
- JTBD 2.0: Bottom-up, data-emerged job statements

## Core Tenets

1. Start with user data, not frameworks
2. Let job statements emerge from evidence
3. Validate every statement against multiple sources
4. Distinguish between functional, social, and emotional dimensions
5. Track frequency and confidence of each insight

## The 5T Principles (see jtbd_quality_checker)

- True: Is this really what the user wants?
- Tacit: Is this unspoken/implied?
- Touch: Does it feel personally relevant?
- Tension: Is there frustration or friction?
- Trigger: What event activates this need?

## Evidence Hierarchy

1. Direct quotes from multiple users (strongest)
2. Observed behavior patterns
3. Single user direct quotes
4. Inferred patterns across users
5. Secondary research (weakest)
