# Kano Model Integration

## Purpose

Classify user needs into categories that predict satisfaction impact.

## Categories

- **Must-be (Basic)**: Expected features. Absence causes dissatisfaction.
- **One-dimensional (Performance)**: More is better. Linear satisfaction.
- **Attractive (Delight)**: Unexpected. Causes delight if present, no dissatisfaction if absent.
- **Indifferent**: Doesn't affect satisfaction.
- **Reverse**: Presence causes dissatisfaction.

## Application to JTBD

Map desired outcomes to Kano categories to prioritize:
- Must-be outcomes: Minimum viable job completion
- Performance outcomes: Competitive differentiation
- Attractive outcomes: Innovation and delight

## Interview Questions

- How would you feel if [outcome] was achieved?
- How would you feel if [outcome] was NOT achieved?
- Pair responses to classify (functional/dysfunctional)
