# SaaS Product Case Study

## Context
15 B2B interviews with project managers evaluating team collaboration tools

## Data Quality
- Source: 15 user interviews (SaaS buyers)
- Sample: PMs and team leads, 5-50 person teams
- Time: Cross-section over 3 months

## Key JTBDs

### JTBD 1: Cross-team visibility
When: Managing multiple projects across departments
I want to: See real-time status of all work without asking for updates
So I can: Make informed resource allocation decisions

### JTBD 2: Onboarding new members
When: New team members join mid-project
I want to: Get them up to speed without hours of one-on-one meetings
So I can: Maintain team velocity

## Need Analysis
- Barrier: Information scattered across tools
- Risk: Missing critical deadlines
- Desired: Centralized but role-filtered visibility

## Product Translation
JTBD 1 -> Requirement: Real-time cross-project dashboard
-> Feature: Executive overview with drill-down
-> User Story: As a PM, I want to see all project statuses on one screen, so I can identify bottlenecks at a glance
