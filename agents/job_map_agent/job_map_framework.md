# Job Map Framework

## Reading the Job Map

The job map reads from Define to Conclude, but should be built starting from Execute.

## For each step, identify:

1. **Input**: What is needed to start this step?
2. **Action**: What does the user actually do?
3. **Output**: What is produced/completed?
4. **Decision Point**: Does the user have to choose?
5. **Pain Point**: What is frustrating or difficult?

## Identifying Job Map Steps from Data

Look for these indicators in user quotes:
- "First I need to..." -> Define/Locate
- "Then I set up..." -> Prepare
- "I double-check..." -> Confirm
- "I do the actual..." -> Execute
- "I keep an eye on..." -> Monitor
- "If something goes wrong..." -> Modify
- "Finally I..." -> Conclude
