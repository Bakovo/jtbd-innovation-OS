# Switch Timeline Framework

Stage 1: First Thought
- What event/issue first made user consider change?
- Often a specific pain point or trigger event

Stage 2: Passive Observation
- User notices alternatives without actively seeking
- Social influence, ads, friends' recommendations

Stage 3: Active Search
- A specific trigger event activates search
- User now actively looking for alternatives

Stage 4: Comparison
- User evaluates 2-3 options
- What criteria matter most?

Stage 5: Decision
- Final deciding factor
- Often emotional + rational

Stage 6: Experience
- Did the new solution deliver?
- Any regrets or surprises?
