# Data Quality Assessment Logic

## Quality Dimensions

1. **Authenticity**: Is the data from real users?
2. **Recency**: Is the data current enough?
3. **Relevance**: Does the data relate to a job?
4. **Richness**: Are there enough details for analysis?
5. **Representativeness**: Does it cover diverse user types?

## Bias Detection Checklist

- [ ] Self-report bias: Users claim what sounds good
- [ ] Recency bias: Users recall most recent experience
- [ ] Selection bias: Only certain user types captured
- [ ] Moderator bias: Questions led users to certain answers
- [ ] Cultural bias: All users from similar background

## Data Risk Levels

- Low: Multiple sources, 10+ users, rich descriptions
- Medium: Single source, 5-9 users, moderate detail
- High: Single source, <5 users, thin descriptions
