# Observation Analysis Logic

## Say-Do Gap Detection Patterns

### Pattern 1: Aspiration vs Behavior
User says they want X but does Y
-> This reveals what they think they should want vs what works for them

### Pattern 2: Claim vs Workaround
User says current solution is fine but describes extensive workarounds
-> Reveals hidden friction they've normalized

### Pattern 3: Explicit vs Implicit Priorities
User says feature X matters but spends more time on Y
-> Reveals true priority hierarchy

### Pattern 4: Stated vs Emotional Response
User says they are satisfied but description reveals frustration
-> Reveals social desirability bias

## Signal Strength

- Strong: Gap confirmed across multiple users
- Moderate: Gap seen in 2-3 users or single user with multiple examples
- Weak: Single instance or inferred gap
