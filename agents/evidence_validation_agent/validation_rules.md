# Validation Rules

## Evidence Strength

| Strength | Definition |
|----------|------------|
| Strong | 3+ users, direct quotes, consistent pattern |
| Moderate | 2 users or 1 user with multiple examples |
| Weak | 1 user, single mention, inferred |

## 5T Verification Questions

True: "Is this really about the user's progress, or is this an AI inference?"
Tacit: "Did the user say this explicitly, or is this between the lines?"
Touch: "Does this feel personally real, or is it generic?"
Tension: "Is there evidence of struggle, frustration, or desire?"
Trigger: "Can we point to a specific event that starts this job?"

## Rejection Criteria

Reject insights that:
- Have no supporting evidence
- Are clearly product/feature suggestions from users
- Are contradictory to evidence without explanation
- Are generic statements applicable to any product
