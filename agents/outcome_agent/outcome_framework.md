# Desired Outcome Engineering Framework

## Quality Criteria

A good desired outcome is:
1. **Measurable**: Can be quantified or observed
2. **Controllable**: Within the scope of the job
3. **Solution-independent**: Does not imply a specific solution
4. **Contextual**: Specifies when and where
5. **Stable**: Not likely to change with technology

## Direction Selection Guide

- Eliminate: When users express frustration/pain
- Reduce: When users express dissatisfaction with current level
- Increase: When users express desire for more
- Maintain: When users express concern about losing something
- Achieve: When users express aspiration

## BAD vs GOOD Outcomes

BAD: Make it easier to find products
GOOD: Minimize the time spent searching for products with the right specifications when shopping for electronics online

BAD: Improve the user experience
GOOD: Eliminate the confusion users feel when comparing pricing tiers during the checkout process
